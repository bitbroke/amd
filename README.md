# 🛡️ PrivacyLens: Neuro-Symbolic Privacy Audit System

PrivacyLens is a Neuro-Symbolic Alignment System designed to quantify the gap between a software's declared legal privacy policy and its actual binary execution on a Windows system. 

Just as `git diff` shows code changes, PrivacyLens shows **Behavioral Drift**. It treats the Privacy Policy as the "Spec" and the Binary Execution as the "Implementation," identifying exactly where the software deviates from its legal claims.

---

## 🧠 How It Works (The Architecture)

The system follows a 3-phase parallel processing pipeline that converges at the Alignment Engine:

1. **Phase 1: The Detective (Static Analysis)**
   * Parses the target `.exe` using `pefile`.
   * Extracts static capabilities (APIs) and hidden strings (Hardcoded URLs, IPv4 addresses, Base64 payloads).
   
2. **Phase 2: The Verifier (Dynamic OS-Observation)**
   * Launches the target application in a sandbox environment.
   * Acts as a "Security Camera" using `psutil` to monitor the host OS.
   * Captures real-time file access (`DYNAMIC_FILE`) and network connections (`DYNAMIC_NET`) without requiring heavy VM instrumentation or code injection.

3. **Phase 3: The Judge (Neuro-Symbolic Alignment)**
   * **The Lawyer (NLP):** Uses `cross-encoder/nli-distilroberta-base` to perform Natural Language Inference (NLI) on the provided Privacy Policy text.
   * **The Alignment:** Compares the extracted Technical Vectors (Static + Dynamic) against the NLP Policy Vectors.
   * Calculates a **Risk-Weighted Verdict** to determine if the application's behavior is legally justified by its policy.
